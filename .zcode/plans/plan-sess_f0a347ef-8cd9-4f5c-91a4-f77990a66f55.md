## Reemplazar `--cpu-moe` por `--n-cpu-moe` en el Optimizador

Son flags distintos: `--cpu-moe` (booleano, **todos** los expertos a CPU) vs `--n-cpu-moe N` (las **primeras N capas** con expertos en CPU). El optimizador debe escribir/leer `--n-cpu-moe`, reubicar el control justo debajo de "Capas en GPU", y mostrarlo **solo** cuando el modelo sea identificable como MoE.

### 1. Backend — `src/script-parser.ts` (parseo del script real)
- Línea 115: cambiar `flagValue(argv, '--cpu-moe')` → `flagValue(argv, '--n-cpu-moe')` (con alias `--cpu-moe` para no romper scripts viejos que ya lo usen: `?? flagValue(argv, '--cpu-moe')`).
- `src/types.ts:70`: actualizar el comentario del campo `nCpuMoe` a `--n-cpu-moe`.
- El flag se aplica al script **desde el frontend** (vía flag-writer), así que el backend solo lo parsea; `router.ts` (línea 274) ya lee `body.params?.nCpuMoe ?? parsed.nCpuMoe` — sin cambios.

### 2. Frontend — `flag-writer.ts` (escribir/leer el flag en el script)
- **`parseParamsFromScript`** (línea 171): `flagNum(tokens, '--cpu-moe')` → `flagNum(tokens, '--n-cpu-moe', ['--cpu-moe'])` (alias para compat retro).
- **`applyTunedParams`** (líneas 208-213): `setFlagValue/removeFlag(tokens, '--cpu-moe', ...)` → usar `--n-cpu-moe`.
- Comentario del header del módulo (línea 13): actualizar la lista de flags gestionados.
- **Importante**: al cambiar de flag, `removeFlag(tokens, '--n-cpu-moe')` debe limpiar también el `--cpu-moe` viejo si quedaba, para no dejarlo duplicado. Añado un `removeFlag(tokens, '--cpu-moe')` antes de setear el nuevo.

### 3. Frontend — `optimizer-modal.ts` (lógica del diálogo)
- **Computed `isMoe`**: nuevo signal derivado `computed(() => isModelMoe(this.meta()))` que detecta MoE por el nombre (`meta().base`): patrones `A\d+B` (p.ej. "35B-A3B"), `MoE`, `Mixtral`, `Nx\d+B`, o presencia de expertos. Reutilizo la regex existente de `format.ts` (`SIZE_RE` con `A\d+B`) extendida.
- Lo defino como función pura `isModelMoe(base: string | null): boolean` en `format.ts` (junto a `parseModel`) para reutilizar y testear.
- Comentario del header del componente (línea 62): `--cpu-moe` → `--n-cpu-moe`.

### 4. Frontend — `optimizer-modal.html` (UI)
- **Mover** el bloque del control MoE (líneas 256-274) para que quede **inmediatamente debajo** del bloque "Capas en GPU (n-gpu-layers)" (líneas 98-116), antes de "Batch size".
- **Envolverlo en `@if (isMoe())`** para que solo aparezca si el modelo es MoE detectable.
- **Label**: `MoE en CPU (--cpu-moe)` → `MOE`.
- **Tooltip**: reemplazar el actual por: *"Indica cuántos expertos (experts) se dejan en la CPU para ahorrar VRAM."*
- El slider mantiene `[min]=0`, `[max]=64`, `[step]=1`, default 0, y el binding `nCpuMoe` (sin renombrar el campo del tipo — sigue siendo `nCpuMoe`).

### 5. Frontend — `core/models/types.ts:185`
- Actualizar el comentario del campo `nCpuMoe` a `--n-cpu-moe`.
