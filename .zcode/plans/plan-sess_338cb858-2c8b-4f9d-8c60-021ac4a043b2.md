## Calibración real del optimizador (dry-fit)

Reemplazar la calibración manual (offset) por una **calibración real** que arranca `llama-server` con el script actual, espera a que el modelo cargue (`m.ready`), mide la VRAM consumida de verdad (delta de `--list-devices`, igual que el benchmark) y detiene el servidor. El resultado reemplaza al estimado en la barra principal del modal.

### Parte 1 — Eliminar la calibración manual (offset) por completo

Revertir todo lo añadido en el turno anterior:
- **Backend**: borrar `VRAM_OFFSET_FILE` de `src/config.ts` y los endpoints `GET/POST /vram-offset` de `src/router.ts` (+ el import).
- **Frontend**: borrar `getVramOffset/saveVramOffset` de `llama-bench.service.ts`; borrar `offsetMiB` signal, `offsetLoaded` flag, `adjustOffset/resetOffset` handlers y la carga en `loadDevices` de `optimizer-modal.ts`; borrar la fila "Calibración" del HTML y los estilos `.calib-*` del CSS. Revertir `bars()` y `totalUsedMiB` a su forma sin offset (manteniendo el baseline de VRAM en uso y el bug-fix híbrido, que sí se quedan).

### Parte 2 — Backend: endpoint `POST /dryfit`

- **`src/dryfit.ts`** (nuevo): función `runDryfit(script): Promise<DryfitResponse>`. Recorta el ciclo de `runBenchmark` al mínimo:
  1. `parseScript` + `assertBinaryExists` (errores tempranos → `DryfitResponse` con `error`).
  2. `listDevices(parsed.binary)` → baseline de VRAM libre por device.
  3. `setBenchAbortController(controller)` + `checkAbort` en cada paso (cancelable vía `/benchmark/stop`).
  4. `m = await startServer(parsed)`.
  5. `await Promise.race([m.ready, <abort>])` — si el proceso muere en arranque (OOM, modelo inválido), rechaza al instante. **No** hay `waitForServer` ni request de inferencia.
  6. `finalDevices = await listDevices(...)` → `computeDeviceVram(baseline, final, parsed.device)` = delta real de VRAM consumido por device.
  7. `totalMiB = suma de deviceVram.usedMiB` (fallback a `readGpuStats`/`subtractGpuBaseline` si `deviceVram` está vacío, igual que el render del benchmark).
  8. `loadTimeSeconds` parseado de los logs (`pollMetricsUntilReady` no aplica sin inferencia; usar el parser directo `parseMetricsFromLogs` sobre el slice desde `logStartIndex`, o el delta de timestamps `loading model`→`model loaded` que ya hace el fallback).
  9. `finally { await stopServer(); setBenchAbortController(null) }` — el servidor se detiene SIEMPRE.
  - Devuelve `DryfitResponse { perDevice, totalMiB, loadTimeSeconds, fits, error: null }`. En error devuelve `{ perDevice: [], totalMiB: null, loadTimeSeconds: null, fits: false, error: msg }` (no lanza — el router solo envuelve excepciones inesperadas en 500).
- **`src/router.ts`**: ruta `POST /dryfit` con el MISMO guard de concurrencia que `/benchmark` (`benchmarkRunning` → 409, `managed` → 409). Setea `benchmarkRunning=true` alrededor de la llamada (en `finally` lo limpia). Lee `{ script }` del body. Reusa `DryfitResponse` ya tipado en `src/types.ts:293`.
- Reusa: `startServer/stopServer/urlFor/assertBinaryExists` (server-manager), `listDevices/computeDeviceVram/detectBackend` (devices), `parseScript` (script-parser), `readGpuStats/subtractGpuBaseline` (gpu), `parseMetricsFromLogs` (metrics), `benchmarkRunning/benchAbortController/setBenchAbortController` (state).

### Parte 3 — Frontend: service + modal

- **`llama-bench.service.ts`**: añadir `dryfit(script): Observable<DryfitRequestResponse>` (POST `/dryfit`). Importar el tipo `DryfitRequestResponse` ya declarado en `models/types.ts:250`.
- **`optimizer-modal.ts`**:
  - Signals: `calibrating = signal(false)`, `measured = signal<DryfitResponse | null>(null)`, `calibError = signal<string | null>(null)`.
  - `calibrate()`: setea `calibrating(true)`, limpia `measured`/`calibError`, llama `this.api.dryfit(script)`, en `next` guarda `measured` o `calibError`, en `complete` apaga `calibrating`. Usa el `script()` del store tal cual (el usuario calibra con lo que tiene en el editor).
  - Derived: `displayUsedMiB = computed(() => measured()?.totalMiB ?? totalUsedMiB())` — cuando hay medición real, la barra/número grande la usan; si no, el estimado. Mismo para `displayPct` y `overflow` (comparan `displayUsedMiB` vs capacidad).
  - `deltaMiB = computed(() => measured() ? measured()!.totalMiB - totalUsedMiB() : null)` para la fila de diferencia.
  - Reset `measured`/`calibError` al cerrar el diálogo (en el effect existente) y al `reloadDevices`.
- **`optimizer-modal.html`**:
  - En el `summary-head`: botón "Calibrar ⚡" (`p-button`, severity info, icono `pi-bolt`) que se deshabilita y muestra spinner mientras `calibrating()`. Lo pongo a la izquierda del badge "estimado". Tooltip: "Arranca el modelo y mide la VRAM real consumida (tarda lo que tarda en cargar el modelo)".
  - La barra grande + número pasan a usar `displayUsedMiB`/`displayPct`. El badge cambia: si hay `measured()` muestra "medido" (verde, real) en vez de "estimado".
  - Fila nueva en el desglose (solo si `measured()`): "Real (medido)" con `fmtGB(measured.totalMiB)` y un subtítulo Δ vs estimado (`+0.5 GB` verde si el estimado fue menor, `-0.5 GB` rojo si fue mayor). Per-device: el bloque "Por dispositivo" puede mostrar el `usedMiB` medido cuando hay medición (sobreescribir `bars().usedMiB` con `measured.perDevice[i].usedMiB` si coincide el id).
  - Mensaje de error si `calibError()`.
- **`optimizer-modal.css`**: estilos para el botón de calibrar + badge "medido" + la fila Δ (colores verde/rojo según signo).

### Qué NO cambia
- El endpoint `/estimate` (sigue siendo heurístico, rápido, sin arrancar el binario).
- El bug-fix de modelos híbridos (KV cache con `attentionLayers`).
- El switch "vram disponible" + baseline de VRAM en uso.
- El ciclo completo de `/benchmark`.

### Gotchas a manejar
- `m.ready` se resuelve cuando el servidor escucha, NO necesariamente cuando los slots terminan de alocar KV. Tras `m.ready` puede hacer falta un `sleep` corto (ej. 500ms) antes del `listDevices` final para que la VRAM del ctx quede reservada. Lo incluyo.
- `runDryfit` no lanza en errores esperados (OOM, modelo inválido): devuelve `DryfitResponse.error` poblado → el router responde 200 con `{ok:true, dryfit:{...,error}}` para que el frontend muestre el error inline, no como exception HTTP. Excepciones inesperadas → 500.
- El dryfit ocupa el puerto 8080 del `llama-server` (igual que el benchmark): por eso el guard contra `managed` y `benchmarkRunning`. Si el usuario tiene un servidor manual corriendo, responde 409 con mensaje claro.
